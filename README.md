## How to Install ğŸš€

Open WebUI is now a standalone frontend that connects to any compatible backend you deploy. The bundled Python backend has been removed from this repository.

### Pointing the UI at your backend

- Append a `backend` query parameter when opening the app: `https://your-host/index.html?backend=https://api.example.com`.
- Or set `VITE_WEBUI_BACKEND_URL` when building. The query parameter always wins and is saved in `localStorage` for future visits.

### Local development

1. Install dependencies: `npm install`
2. Start the dev server: `npm run dev -- --host`
3. Open the app with your backend URL: `http://localhost:5173/?backend=http://localhost:8080`

### Production build

1. `npm install`
2. `npm run build`
3. Serve the `build/` directory with any static host or CDN.
4. Provide the backend URL through the `backend` query parameter or by baking `VITE_WEBUI_BACKEND_URL` into the build.

### Docker ğŸ³

The included Dockerfile now builds and serves the static frontend with Nginx. Example:

```bash
docker build -t open-webui-frontend .
docker run -d -p 3000:80 -e VITE_WEBUI_BACKEND_URL=https://api.example.com --name open-webui-frontend open-webui-frontend
```

### Troubleshooting

- If the UI cannot reach your backend, double-check the `backend` query parameter or the `VITE_WEBUI_BACKEND_URL` you baked into the build.
- When hosting behind a path prefix, ensure your static server rewrites to `index.html` so the client-side router can resolve routes.

### Using the Dev Branch ğŸŒ™

The `:dev` tag now publishes only the frontend build. Use it when you want the latest UI changes and point it to the backend of your choice.
